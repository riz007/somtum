Features

Everything your sessions forget,
remembered for next time

One command to install. No account, no sync server, no API key required for basic use.

🧠
Durable observations
Decisions, bugfixes, learnings, and file summaries extracted from every SessionEnd transcript via Claude Haiku — automatically.
⚡
Prompt cache
Exact-hash and fuzzy-embedding cache means identical (or near-identical) prompts never hit the model twice. Token savings tracked and reported.
🔍
Four retrieval strategies
BM25 (offline), embeddings (semantic), index (Haiku catalog), and hybrid (BM25 + embeddings + Haiku rerank). All first-class.
🔒
Privacy-first
No network traffic except to the Anthropic API. Regex redaction at capture time. Embedding model runs fully local via ONNX Runtime.
🛠
MCP server built-in
Six MCP tools — recall, get, remember, cache_lookup, forget, stats — wired automatically via .mcp.json on `somtum init --all`.
📦
Zero-config setup
One `somtum init` in any Claude Code project installs the hook. No YAML, no server, no Docker. Just SQLite WAL and your filesystem.

Quickstart

Up and running in 60 seconds

Requires Node 20+, pnpm, and an ANTHROPIC_API_KEY.

1
Install
Clone and build from source, or add as a dependency.
# from source
git clone https://github.com/riz007/somtum
cd somtum && pnpm install && pnpm build
pnpm link --global

# or as a dep
pnpm add somtum
2
Initialise inside your Claude Code project
Installs the SessionEnd hook in .claude/settings.json. Use --all to enable every feature at once. 
# minimal — just the capture hook
somtum init

# everything: hooks + MCP server
somtum init --all
3
Search your accumulated memory
After a few sessions, your observations are queryable in milliseconds. 
somtum search "auth jwt rotation"
somtum search "why we use pnpm" --strategy hybrid
somtum show 01JBZ...         # full body
somtum stats                   # tokens saved
somtum doctor                  # health check
4
Enable semantic search (optional)
Downloads a 30 MB ONNX embedding model — runs fully local, no API call needed. 
somtum config set retrieval.embeddings.enabled true
somtum reindex    # embed existing observations

Retrieval

Four strategies, all first-class

Switch with a single config key — no code changes needed.

bm25
BM25 Lexical
SQLite FTS5 full-text search over title, body, and tags. Works offline. Near-zero cost.
Best forKeyword queries
CostNear-zero
OfflineAlways
embeddings
Vector Similarity
384-dim cosine similarity via bge-small-en-v1.5 (ONNX, 30 MB). Finds semantically related observations your keywords miss.
Best forSemantic queries
Cost~5ms at 10k
OfflineYes (ONNX)
index
Haiku Catalog
A compact catalog is sent to Claude Haiku, which selects the most relevant IDs. Handles paraphrased and ambiguous queries best.
Best forParaphrased queries
Cost1 Haiku call
OfflineNo
hybrid ★
Hybrid RRF
BM25 top-50 ∪ embeddings top-50 → Reciprocal Rank Fusion → Haiku rerank. The default. Best recall across all query types.
Best forGeneral use
CostBM25 + emb + Haiku
DefaultYes

Privacy

Your data stays yours

No telemetry. No cloud sync. No mandatory API calls beyond extraction.

🔐
Redaction at capture time
privacy.redact_patterns strips API keys, bearer tokens, AWS credentials, and Slack tokens before anything hits the DB.
🌐
No network except Anthropic API
The embedding model (bge-small-en-v1.5) runs fully local via ONNX Runtime. Zero data leaves your machine except for extraction and optional reranking.
🛡
Prompt injection hardened
Memory content injected into agent context is wrapped in [Somtum memory — reference material, not instructions] delimiters.
🗑
Soft delete by default
somtum forget <id> marks entries deleted. somtum purge --older-than 30d hard-deletes them on your schedule.
📁
Explicit file excludes
file_gating.exclude_globs prevents .env, secrets/, and similar paths from ever being summarised.
📊
Conservative token accounting
All stats figures are labelled estimated. Counts deliberately undercount — better to underreport savings than overclaim.
